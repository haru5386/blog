#### 電腦可讀取單位

- 為什麼電腦只能看得懂二進位？
  電腦的底層硬體其實很簡單，它只會判斷「有電（1）」或「沒電（0）」。所以電腦世界裡，不管是圖片、聲音、文字或影片，通通都要先變成一串 0 跟 1，也就是「二進位（Binary）」的形式，電腦才能理解。

- 那十六進位是什麼？
  十六進位（Hexadecimal）是給「人類」用的，因為直接看一長串 0 和 1 很難讀懂。舉個例子：

  ```r
  二進位：0b1111 1111
  十進位：255
  十六進位：0xFF
  ```

  是不是十六進位好讀多了

#### 進位表示方式

1. **二進位（Binary）**

   **二進位用 `0b` 作為前綴。**例如：0b1010 表示二進位數字 1010，即十進位數字 10。

   在 JavaScript 和 Python 等編程語言中，0b 被用來標識二進位數字。

   例子：
   0b1010 = 10（十進位）

2. **十進位（Decimal）**

   **十進位是最常見的數字系統，沒有特別的前綴，**直接寫數字即可。例如：42 表示十進位數字 42。

3. **十六進位（Hexadecimal）**
   **十六進位使用 `0x` 作為前綴來表示。**例如：0xA 表示十六進位數字 A，即十進位數字 10。

   例子：
   0xA = 10（十進位）

#### 基本單位認識

1. **位元(bit)**：

   bit 是電腦中資訊的最小單位，只能是 0 或 1。
   8 個 bit 就可以表示從 0 到 255 的數字範圍（也就是 2⁸ = 256 種組合）。

   ```r
   00000000 （最小） → 十進位 0
   11111111 （最大） → 十進位 255
   ```

2. **位元組(byte)**：

   1 byte = 8 bits。
   所以很多資料的單位都會用 KB、MB、GB，其實就是「一堆 bytes」的意思。

3. **常見進位制**：

- 十進位（Decimal）：人類日常最常用的系統，每位是 10 的倍數。
  例如：253 = 2×100 + 5×10 + 3×1

- 十六進位（Hex）：每位是 16 的倍數，數字從 09，再加上 AF 代表 10~15。

  ```r
      A = 10
      B = 11
      C = 12
      D = 13
      E = 14
      F = 15
  ```

#### 進位制轉換教學

##### 十六進位轉換成十進位

以 `FF` 為例：

    ```r
    F   F
    ↑   ↑
    15  15
    ```
    計算方式是：
    ```r
    F × 16¹ + F × 16⁰
    = 15 × 16 + 15 × 1
    = 240 + 15
    = 255 ✅
    ```

💡 **Tips**

`FF` 是兩位十六進位的最大值，所以 → `FF = 16² - 1 = 256 - 1 = 255`  
這也是為什麼 **8-bit 二進位最大值是 11111111 = 255** 對應的十六進位是 FF

###### 二進位轉換成十六進位：

快速轉換三步驟：

    1. **從右邊開始，每 4 個 bit 一組分組**
        - 因為 2⁴ = 16，所以 4 位元的二進位剛好對應 1 個十六進位位數
    2. **每組轉成十進位，再對照十六進位的符號**
    3. **不足 4 位的左邊補 0**

    **舉例**
    ```yaml
    二進位：11010110
    分組：1101 0110
    轉換：
    - 1101 = 13 → D
    - 0110 = 6 → 6

    ✅ 最後結果 → 0xD6
    ```

### 那為什麼我們需要了解 bit / byte / 進位制？

**因為我們處理的資料、文字、圖檔、網路請求、前後端溝通，全都是 bit 的排列組合！**
| 前端情境 | 為什麼需要 bit/byte/進位制 概念|
| :------ | :--------- |
| 傳 emoji、中文字、特殊符號 | 要考慮 UTF-8 編碼、bytes 數量、轉義|
| base64 圖片/檔案上傳 base64 | 會把二進位轉成文字，大小膨脹 1.33 倍|
| 檔案大小、限制、壓縮| 單位轉換：KB、MB、GB，其實是 bytes|
| 操作 canvas、音訊、ArrayBuffer| 資料通常是二進位結構（typed array）|
| 設計網路協議、API（尤其 IoT）| 傳遞結構資料時用 bit-field 節省空間|
| 轉色碼（#FFAA00）| 每個 16 進位數字都是 4 bit，跟 RGB 有關|
| 儲存 JSON / 文字 | 瞭解為什麼 A 只佔 1 byte，但 emoji 佔 4 bytes|

### 結論：

電腦不懂中文、英文，也不懂圖片，它只懂「有沒有電」——也就是 0 跟 1。
從這兩個簡單的數位開始，我們用不同的進位系統（像是二進位、十六進位）來「翻譯」資訊，讓電腦和人都能理解彼此。

理解這一點，不只幫助你更懂電腦怎麼運作，也能打下更穩固的工程基礎！
